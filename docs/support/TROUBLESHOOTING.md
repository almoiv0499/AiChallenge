# Руководство по устранению неполадок

## Проблемы с авторизацией

### Ошибка OAUTH_INVALID_TOKEN

**Описание:** При попытке входа через OAuth провайдера (Google, GitHub, etc.) появляется ошибка "Invalid OAuth token".

**Причины:**
1. Истёкшие cookies или сессия
2. Конфликт между несколькими аккаунтами провайдера
3. Блокировка запросов расширениями браузера
4. Проблемы с временем на устройстве
5. Корпоративный прокси/файрвол

**Решение:**
```
Шаг 1: Очистите cookies
- Chrome: Настройки → Конфиденциальность → Очистить данные → Cookies
- Firefox: Настройки → Конфиденциальность → Очистить данные

Шаг 2: Выйдите из всех аккаунтов провайдера
- Перейдите на accounts.google.com и выйдите
- Закройте все вкладки браузера
- Откройте новую вкладку

Шаг 3: Попробуйте режим инкогнито
- Откройте новое окно в режиме инкогнито
- Попробуйте войти снова

Шаг 4: Проверьте расширения
- Отключите все расширения браузера
- Особенно: блокировщики рекламы, VPN расширения

Шаг 5: Проверьте системное время
- Убедитесь, что время и дата на устройстве корректны
- Включите автоматическую синхронизацию времени
```

### Ошибка SESSION_EXPIRED

**Описание:** Сессия пользователя неожиданно завершается.

**Решение:**
1. Проверьте настройки cookies в браузере - они должны быть разрешены
2. Отключите режим "Удалять cookies при закрытии браузера"
3. Убедитесь, что время на устройстве синхронизировано

## Проблемы с оплатой

### Ошибка PAYMENT_DECLINED

**Описание:** Платёж отклонён платёжной системой.

**Возможные причины:**
- Недостаточно средств на карте
- Карта заблокирована для онлайн-платежей
- Превышен лимит операций
- Банк заблокировал подозрительную транзакцию
- Неверные данные карты

**Решение:**
```
1. Проверьте баланс карты
2. Позвоните в банк и уточните:
   - Не заблокирована ли карта для интернет-платежей
   - Нет ли ограничений на платежи за рубеж
3. Попробуйте другую карту
4. Используйте альтернативный способ оплаты (PayPal, криптовалюта)
```

### Ошибка CARD_3DS_FAILED

**Описание:** Не пройдена 3D-Secure проверка.

**Решение:**
1. Убедитесь, что 3D-Secure подключен к вашей карте
2. Проверьте, приходят ли SMS от банка
3. Попробуйте использовать другой браузер
4. Убедитесь, что номер телефона в банке актуален

## Технические проблемы

### Краш при запуске (Android)

**Диагностика:**
```
1. Соберите информацию:
   - Версия Android
   - Модель устройства
   - Версия приложения
   - Когда началась проблема

2. Проверьте логи:
   - Настройки → О приложении → Отправить логи
```

**Решение по приоритету:**
```
Уровень 1 - Базовые действия:
- Перезагрузите устройство
- Очистите кэш приложения
- Обновите приложение до последней версии

Уровень 2 - Расширенные действия:
- Очистите данные приложения (внимание: потеря локальных данных!)
- Удалите и переустановите приложение
- Проверьте свободную память (нужно минимум 500 МБ)

Уровень 3 - Для разработчиков:
- Соберите crash dump через adb logcat
- Проверьте совместимость с кастомными ROM
```

### Проблемы синхронизации

**Диагностика sync_failed:**
```
1. Проверьте версии приложений на всех устройствах
2. Убедитесь в стабильности интернет-соединения
3. Проверьте статус серверов: status.example.com
4. Выполните принудительную синхронизацию
```

**Коды ошибок синхронизации:**

| Код | Описание | Решение |
|-----|----------|---------|
| SYNC_001 | Нет подключения | Проверьте интернет |
| SYNC_002 | Таймаут сервера | Попробуйте позже |
| SYNC_003 | Конфликт версий | Обновите приложение |
| SYNC_004 | Превышен лимит | Дождитесь сброса лимита |
| SYNC_005 | Повреждены данные | Переустановите приложение |

## API ошибки

### Ошибка 401 Unauthorized
- Проверьте, не истёк ли API ключ
- Убедитесь, что ключ передаётся в заголовке Authorization
- Проверьте, включён ли API доступ на вашем тарифе

### Ошибка 429 Too Many Requests
- Превышен лимит запросов
- Используйте экспоненциальный backoff
- Рассмотрите переход на Enterprise тариф

### Ошибка 500 Internal Server Error
- Проверьте статус серверов: status.example.com
- Попробуйте повторить запрос через 5 минут
- Сообщите в поддержку, если проблема сохраняется
