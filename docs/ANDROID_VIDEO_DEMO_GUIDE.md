# Подсказки для записи видео-демо Android приложения

## Структура видео (рекомендуемый порядок)

### 1. Введение (30 секунд)

- Показать общий вид приложения
- Кратко описать что это: "Android приложение для чата с AI ассистентом"

### 2. Архитектура проекта (1-2 минуты)

**Что показать:**
- Структуру модулей в Android Studio:
  - Модуль `app` (UI)
  - Модуль `chatAI` (бизнес-логика)
- Gradle файлы, показать зависимости
- Модульность: подчеркнуть, что `chatAI` не зависит от Android UI

**Что сказать:**
- "Приложение использует модульную архитектуру"
- "Модуль chatAI содержит всю бизнес-логику и может быть переиспользован"
- "UI слой отделен от логики через MVVM паттерн"

### 3. Структура кода (1-2 минуты)

**Что показать:**
- Структуру пакетов в модуле `chatAI`:
  - `data/` - Room, Repository
  - `domain/` - UseCase, API Service
- Структуру пакетов в модуле `app`:
  - `ui/screen/` - экраны Compose
  - `ui/viewmodel/` - ViewModels
  - `di/` - Koin модуль

**Что сказать:**
- "Используется Clean Architecture с разделением на слои"
- "Data слой - Room база данных и репозиторий"
- "Domain слой - UseCases и API сервисы"
- "UI слой - Jetpack Compose и ViewModels"

### 4. Room база данных (1 минута)

**Что показать:**
- Модель `ChatMessage` с полями: id, role, content, timestamp
- Enum `MessageRole` с ролями: SYSTEM, USER, ASSISTANT, TOOL, MCP
- DAO с методами для работы с БД
- Database класс с настройкой Room

**Что сказать:**
- "Room база данных хранит историю всех сообщений"
- "Поддерживаются роли: system, user, assistant, tool, mcp"
- "История автоматически сохраняется при каждом сообщении"

### 5. MVVM архитектура (1-2 минуты)

**Что показать:**
- ViewModel с StateFlow для состояния
- UseCase для бизнес-логики
- Repository как абстракция над данными
- Flow в DAO для реактивного получения данных

**Что сказать:**
- "ViewModel управляет состоянием UI через StateFlow"
- "UseCase инкапсулирует бизнес-логику отправки сообщений"
- "Repository предоставляет единый интерфейс к данным"
- "Flow обеспечивает реактивное обновление UI"

### 6. Dependency Injection (Koin) (1 минута)

**Что показать:**
- `AppModule` с регистрацией всех зависимостей
- `MainActivity` с инициализацией Koin
- Как зависимости инжектятся в ViewModel

**Что сказать:**
- "Koin используется для Dependency Injection"
- "Все зависимости регистрируются централизованно в AppModule"
- "ViewModel получает UseCase через constructor injection"

### 7. Jetpack Compose UI (2-3 минуты)

**Что показать:**
- Экран `ChatScreen` с LazyColumn для списка сообщений
- Компонент `ChatMessageItem` с различным оформлением для ролей
- Компонент `MessageInput` для ввода сообщений
- Тему приложения с Material Design 3

**Что сказать:**
- "UI полностью написан на Jetpack Compose"
- "Различное оформление для сообщений пользователя и ассистента"
- "Реактивное обновление UI через StateFlow"

### 8. Функциональность приложения (2-3 минуты)

**Что показать:**
- Запуск приложения на эмуляторе/устройстве
- Отправка сообщения пользователя
- Получение ответа от AI
- Прокрутку списка сообщений
- Восстановление истории после перезапуска (закрыть и открыть приложение)

**Что сказать:**
- "Приложение отправляет сообщения в OpenRouter API"
- "Ответы сохраняются в Room базу данных"
- "История автоматически восстанавливается при перезапуске"

### 9. CI/CD (GitHub Actions) (1-2 минуты)

**Что показать:**
- Файл `.github/workflows/android-ci.yml`
- Запуск workflow в GitHub Actions (если есть доступ)
- Скачивание APK из artifacts после успешной сборки

**Что сказать:**
- "GitHub Actions автоматически собирает проект при push"
- "После успешной сборки APK публикуется как artifact"
- "APK можно скачать из раздела Actions в GitHub"

### 10. Сборка проекта локально (1 минута)

**Что показать:**
- Команду `./gradlew :app:assembleDebug`
- Результат сборки - APK файл
- Расположение APK: `android/app/build/outputs/apk/debug/app-debug.apk`

**Что сказать:**
- "Для локальной сборки используется Gradle"
- "APK находится в папке build/outputs/apk/debug"
- "Можно установить на устройство через `./gradlew :app:installDebug`"

### 11. Заключение (30 секунд)

**Что сказать:**
- Подвести итог: что реализовано
- Упомянуть ключевые технологии: MVVM, Compose, Room, Koin, Ktor
- Подчеркнуть модульность и production-ready код

## Важные моменты для записи

### Технические детали

1. **Скорость:** не торопитесь, но и не затягивайте
2. **Четкость:** говорите понятно, объясняйте термины
3. **Фокус:** выделяйте важные части кода подсветкой курсора
4. **Контекст:** показывайте связи между компонентами

### Визуальные подсказки

1. **Zoom:** увеличивайте код для важных частей
2. **Навигация:** показывайте переходы между файлами
3. **Запуск:** покажите реальный запуск приложения
4. **Интерактивность:** отправьте несколько сообщений

### Что обязательно показать

✅ Модульную структуру (app + chatAI)
✅ Room базу данных с ролями сообщений
✅ MVVM архитектуру (ViewModel → UseCase → Repository)
✅ Jetpack Compose UI
✅ Восстановление истории при перезапуске
✅ CI/CD pipeline (файл или запуск)
✅ Локальную сборку проекта

### Что можно пропустить (если ограничено время)

⏭️ Детали ProGuard правил
⏭️ Подробности темы Material Design
⏭️ Все UseCases (показать только основные)
⏭️ Детали обработки ошибок (можно упомянуть кратко)
